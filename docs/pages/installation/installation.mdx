The guides in this section show you how to install Teleport on your system. As
we explain in [Core Concepts](../core-concepts.mdx), Teleport is a distributed
system with several components working together across your infrastructure. How
you install Teleport depends on:

- Whether you have a Teleport Cloud account or are self-hosting the Teleport
  Auth Service and Proxy Service
- How you choose to deploy Teleport Agents, `tbot`, and any self-hosted plugins
  like the Event Handler and Access Request plugins
- If you are self-hosting, how you choose to deploy the Teleport Auth Service
  and Proxy Service 

## Client tools

If you want to download client tools to access Teleport from your workstation,
visit [Download Client Tools](https://goteleport.com/download/client-tools/).
This page also includes client tool downloads for Windows, which does not
support the `teleport` daemon.

## Teleport cluster setup

The easiest way to get started with Teleport is to [sign
up](https://goteleport.com/signup/) for a Teleport Cloud account. From there,
you can [install Teleport Agents](agents/agents.mdx) so you can enroll your
infrastructure with Teleport.

For instructions on installing a `teleport` binary on a single machine, see
[Install Teleport on a Single Machine](single-machine/single-machine.mdx). The
Teleport Auth Service, Teleport Proxy Service, and Teleport Agent services all
share a binary, though you will need to create a configuration file for each
instance and configure them to communicate with one another as a cluster.

## Self-hosted Teleport clusters

You can read about setting up a self-hosted cluster in [Deploy a
Cluster](../zero-trust-access/deploy-a-cluster/deploy-a-cluster.mdx). We
recommend [reaching out to us directly](https://goteleport.com/contact-us/) to
learn more about getting started with this deployment option and start planning
your approach. Self-hosted Teleport Enterprise deployments typically take place
in conversation with the team at Teleport.
