This chart deploys `tbot` as a DaemonSet for issuing SPIFFE SVIDs to workloads
running on the same node. It configures the SPIFFE Workload API via a Unix
domain socket. This socket can then be mounted into pods for running workloads
to allow them to receive SPIFFE SVIDs.

To use the chart, follow steps 1 and 2 of the [Deploying `tbot` on Kubernetes
guide](../../machine-workload-identity/deployment/kubernetes.mdx) to create a
Bot and join token, then set the following values at minimum:

- `clusterName`: the name of your Teleport cluster
- `teleportProxyAddress`: the address of your Teleport Proxy Service
- `token`: the name of the join token you created
- `workloadIdentitySelector.name`: the name of the WorkloadIdentity resource to
  use when issuing SVIDs
