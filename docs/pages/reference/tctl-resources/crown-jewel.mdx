Represents a Crown Jewel resource. Crown Jewel is a resource that represents a set of resources that are considered critical to the organization. Access Graph uses Crown Jewel to generate audit events if access to a resource has changed.

Example:

```yaml
kind: "string"
sub_kind: "string"
version: "string"
metadata: # See description
spec: # [...]
```
|Field Name|Description|Type|
|---|---|---|
|version|The version of the resource being represented.|string|
|sub_kind|Mandatory field for all resources. Not populated for this resource type.|string|
|spec|The crown jewel spec.|[Crown Jewel Spec](#crown-jewel-spec)|
|metadata|Common metadata that all resources share.||
|kind|The kind of resource represented.|string|

## AWSMatcher

Represents a matcher for AWS resources. Those matchers are used only by Access Graph. Teleport related matchers are defined in the TeleportMatcher.


Example:

```yaml
types: 
  - "string"
  - "string"
  - "string"
regions: 
  - "string"
  - "string"
  - "string"
tags: 
  - # [...]
  - # [...]
  - # [...]
arns: 
  - "string"
  - "string"
  - "string"
```

|Field Name|Description|Type|
|---|---|---|
|types|AWS database types to match, "ec2", "rds", "s3", etc|[]string|
|tags|AWS resource Tags to match. labels is a set of labels.|[][AWSTag](#awstag)|
|regions|AWS regions to query for resources.|[]string|
|arns|ARNs are AWS resources ARN to match.|[]string|

## Teleport Matcher

Represents a matcher for Teleport resources.


Example:

```yaml
kinds: 
  - "string"
  - "string"
  - "string"
labels: 
  - # See description
  - # See description
  - # See description
names: 
  - "string"
  - "string"
  - "string"
```

|Field Name|Description|Type|
|---|---|---|
|names|The name of resources. When the name is provided, it will match resources with the same name.|[]string|
|labels|A set of labels.|[]|
|kinds|Kind is the kind of the resource: ssh, k8s, db, etc Multiple kinds can be provided to match multiple kinds.|[]string|
