## Global Notification Spec

The global notification's specification.


Example:

```yaml
: # [...]
match_all_conditions: true
notification: # [...]

  - "string"
  - "string"
  - "string"
```

|Field Name|Description|Type|
|---|---|---|
|notification|Notification is the notification itself.|[Notification](#notification)|
|match_all_conditions|Match_all_conditions is whether or not all the conditions specified by the matcher must be met, if false, only one of the conditions needs to be met.|Boolean|
|exclude_users|Exclude_users is a list of usernames of users who should never match this notification under any circumstances.|[]string|
|Matcher|For determining the target of this notification.  Types that are valid to be assigned to Matcher:  	*GlobalNotificationSpec_ByPermissions 	*GlobalNotificationSpec_ByRoles 	*GlobalNotificationSpec_All 	*GlobalNotificationSpec_ByUsers|[is Global Notification Spec_Matcher](#is-global-notification-spec_matcher)|

## Notification Spec

The notification specification.


Example:

```yaml
created: # See description
unscoped: true
username: "string"
```

|Field Name|Description|Type|
|---|---|---|
|username|Username is the username of the target user if this is a user-specific notification. Requests for global notifications with a username will be rejected.|string|
|unscoped|Unscoped is whether the notification shouldn't be restricted to a specific audience. This is to prevent the potential future possibility that a user-specific notification contains information that the user should no longer be allowed to see. Default is true.|Boolean|
|created|Created is when the notification was created, in UNIX time.||
