## Cluster Networking Config Spec

The actual data we care about for ClusterNetworkingConfig.


Example:

```yaml
client_idle_timeout: # [...]
keep_alive_interval: # [...]

```

|Field Name|Description|Type|
|---|---|---|
|web_idle_timeout|Sets global cluster default setting for the web UI idle timeouts.|[Duration](#duration)|
|tunnel_strategy|V1 determines the tunnel strategy used in the cluster.|[Tunnel Strategy](#tunnel-strategy)|
|ssh_dial_timeout|A custom dial timeout used when establishing SSH connections. If not set, the default timeout of 30s will be used.|[Duration](#duration)|
|session_control_timeout|The session control lease expiry and defines the upper limit of how long a node may be out of contact with the auth server before it begins terminating controlled sessions.|[Duration](#duration)|
|routing_strategy|Determines the strategy used to route to nodes. 0 is "unambiguous_match"; 1 is "most_recent".|[Routing Strategy](#routing-strategy)|
|proxy_ping_interval|Defines in which interval the TLS routing ping message should be sent. This is applicable only when using ping-wrapped connections, regular TLS routing connections are not affected.|[Duration](#duration)|
|proxy_listener_mode|Proxy listener mode used by Teleport Proxies. 0 is "separate"; 1 is "multiplex".|[Proxy Listener Mode](#proxy-listener-mode)|
|keep_alive_interval|The interval at which the server sends keep-alive messages to the client.|[Duration](#duration)|
|keep_alive_count_max|The number of keep-alive messages that can be missed before the server disconnects the connection to the client.|number|
|idle_timeout_message|The message sent to the user when a connection times out.|string|
|client_idle_timeout|Sets global cluster default setting for client idle timeouts.|[Duration](#duration)|
|case_insensitive_routing|Causes proxies to use case-insensitive hostname matching.|Boolean|
|assist_command_execution_workers|Determines the number of workers that will execute arbitrary Assist commands on servers in parallel|number|
