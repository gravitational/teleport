Represents a notification item.

Example:

```yaml
kind: "string"
sub_kind: "string"
version: "string"
metadata: # See description
spec: # [...]
```
|Field Name|Description|Type|
|---|---|---|
|version|Version is the resource version.|string|
|sub_kind|Sub_kind represents the unique kind of notification this is, eg. `access-request-approved`|string|
|spec|Spec is the notification specification.|[Notification Spec](#notification-spec)|
|metadata|Metadata is the notification's metadata. This contains the notification's labels, and expiry. All custom notification metadata should be stored in labels.||
|kind|Kind is the resource kind ("notification").|string|

## Notification Spec

The notification specification.


Example:

```yaml
created: # See description
unscoped: true
username: "string"
```

|Field Name|Description|Type|
|---|---|---|
|username|Username is the username of the target user if this is a user-specific notification. Requests for global notifications with a username will be rejected.|string|
|unscoped|Unscoped is whether the notification shouldn't be restricted to a specific audience. This is to prevent the potential future possibility that a user-specific notification contains information that the user should no longer be allowed to see. Default is true.|Boolean|
|created|Created is when the notification was created, in UNIX time.||
