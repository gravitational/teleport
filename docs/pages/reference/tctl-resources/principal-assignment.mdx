## Account Assignment Ref

A minimal link between an AWS Account and a permission set. Optionally contains human-readable metadata describing the target AWS Account and Permission Set.


Example:

```yaml
account_id: "string"
account_name: "string"
permission_set_arn: "string"
permission_set_name: "string"
```

|Field Name|Description|Type|
|---|---|---|
|permission_set_name|Optionally holds the human-readable name of the target AWS Permission Set.|string|
|permission_set_arn|Holds the target Permission Set ARN. This will always be set.|string|
|account_name|Optionally holds the heman-readable name of the target AwS account|string|
|account_id|Holds the AWS account ID for the target AWS account. This will always be set|string|

## Principal Assignment Spec

Holds information about the Account Assignments assigned to the target principal.


Example:

```yaml
principal_type: # [...]
principal_id: "string"
external_id_source: "string"
external_id: "string"
```

|Field Name|Description|Type|
|---|---|---|
|principal_type|PrincipalId identifies what kind of principal this assignment applies to, either a User or a Group (i.e. AccessList)|[Principal Type](#principal-type)|
|principal_id|Identifies the Teleport User or Access List that this assignment applies to|string|
|external_id_source||string|
|external_id|ExternalID identifies the user or group within AWS|string|
