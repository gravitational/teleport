The resource representing a global database object import rule. The import rules govern which database objects are imported from databases. See type teleport.dbobject.v1.DatabaseObject for the description of a database object. For rationale behind this type, see the RFD 151.

## Database Object Import Match

Specifies acceptable object names. Must have at least one non-empty member.


Example:

```yaml
table_names: 
  - "string"
  - "string"
  - "string"
view_names: 
  - "string"
  - "string"
  - "string"
procedure_names: 
  - "string"
  - "string"
  - "string"
```

|Field Name|Description|Type|
|---|---|---|
|view_names|View_names specify the names of the views to match. Optional.|[]string|
|table_names|Table_names specify the names of the tables to match. Optional.|[]string|
|procedure_names|Procedure_names specify the names of the procedures to match. Optional.|[]string|

## Database Object Import Scope

Specifies the object scope. Members are matched independently of each other.


Example:

```yaml
database_names: 
  - "string"
  - "string"
  - "string"
schema_names: 
  - "string"
  - "string"
  - "string"
```

|Field Name|Description|Type|
|---|---|---|
|schema_names|Schema_names specify the names of the schemas to include in the scope. Optional.|[]string|
|database_names|Database_names specify the names of the databases to include in the scope. Optional.|[]string|
