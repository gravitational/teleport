## Agent Auto Update Group

Specifies the update schedule for a group of agents.


Example:

```yaml
name: "string"
days: 
  - "string"
  - "string"
  - "string"
start_hour: 1
wait_hours: 1
```

|Field Name|Description|Type|
|---|---|---|
|wait_hours|Wait_hours after last group succeeds before this group can run. This can only be used when the strategy is "halt-on-failure". This field must be positive.|number|
|start_hour|Start_hour to initiate update|number|
|name|Name of the group|string|
|days|Days when the update can run. Supported values are "Mon", "Tue", "Wed", "Thu", "Fri", "Sat", "Sun" and "*"|[]string|

## Auto Update Config Spec Agents

Encodes the parameters of automatic agent updates.


Example:

```yaml
mode: "string"
strategy: "string"
maintenance_window_duration: # See description
schedules: # [...]
```

|Field Name|Description|Type|
|---|---|---|
|strategy|Strategy to use for updating the agents.|string|
|schedules|Schedules specifies schedules for updates of grouped agents.|[Agent Auto Update Schedules](#agent-auto-update-schedules)|
|mode|Mode specifies whether agent autoupdates are enabled, disabled, or paused.|string|
|maintenance_window_duration|Maintenance_window_duration is the maintenance window duration. This can only be set if `strategy` is "time-based". Once the window is over, the group transitions to the done state. Existing agents won't be updated until the next maintenance window.||
