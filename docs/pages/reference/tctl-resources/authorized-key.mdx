The `AuthorizedKey` message represents an authorized key entry for a specific local user. These authorized keys are generated by the server when a particular SSH AuthorizedKey is granted access to a user on the node.

Example:

```yaml
metadata: # See description
kind: "string"
sub_kind: "string"
version: "string"
spec: # [...]
```

## Authorized Key Spec

The authorized key spec.


Example:

```yaml
host_id: "string"
key_fingerprint: "string"
host_user: "string"
key_comment: "string"
key_type: "string"
```

|Field Name|Description|Type|
|---|---|---|
|key_type|Key_type is the ssh's key type.|string|
|key_fingerprint|Key_fingerprint is the SHA256 SSH public key fingerprint.|string|
|key_comment|Key_comment is the authorized key's comment. Authorized keys consist of the following space-separated fields: options, keytype, base64-encoded key, comment.  The options field is optional.|string|
|host_user|Host_user is the user who can be accessed using the fingerprint above.|string|
|host_id|Host_id is the node identifier and must match the credentials used.|string|
