## LDAP Config

The LDAP connection parameters.

Example:

```yaml
addr: "string"
domain: "string"
username: "string"
sid: "string"
insecure_skip_verify: true
server_name: "string"
der_ca_file: "string"
ldap_ca_cert: "string"
enabled: true
site: "string"
```
|Field Name|Description|Type|
|---|---|---|
|addr|The host:port of the LDAP server (typically port 389).|string|
|der_ca_file|The filepath to an optional DER encoded CA cert to be used for verification (if InsecureSkipVerify is set to false).|string|
|domain|The ActiveDirectory domain name.|string|
|enabled|Will automatically locate the LDAP server using DNS SRV records. When enabled, Domain must be set, Addr will be ignored https://ldap.com/dns-srv-records-for-ldap/|Boolean|
|insecure_skip_verify|Decides whether whether we skip verifying with the LDAP server's CA when making the LDAPS connection.|Boolean|
|ldap_ca_cert|An optional PEM encoded CA cert to be used for verification (if InsecureSkipVerify is set to false).|string|
|server_name|The name of the LDAP server for TLS.|string|
|sid|The Security Identifier for the service account specified by Username.|string|
|site|An LDAP site to locate servers from a specific logical site. https://learn.microsoft.com/en-us/openspecs/windows_protocols/ms-adts/b645c125-a7da-4097-84a1-2fa7cea07714#gt_8abdc986-5679-42d9-ad76-b11eb5a0daba|string|
|username|For LDAP authentication.|string|


## LDAP Discovery Config

LDAP discovery configuration for windows desktop discovery service.

Example:

```yaml
base_dn: "string"
filters: 
  - "string"
  - "string"
  - "string"
labels: 
  "string": "string"
  "string": "string"
  "string": "string"
label_attributes: 
  - "string"
  - "string"
  - "string"
rdp_port: 1
```
|Field Name|Description|Type|
|---|---|---|
|base_dn|The base DN to search for desktops. Use the value '*' to search from the root of the domain, or leave blank to disable desktop discovery.|string|
|filters|Additional LDAP filters to apply to the search. See: https://ldap.com/ldap-filters/|[]string|
|label_attributes|LDAP attributes to apply to hosts discovered via LDAP. Teleport labels hosts by prefixing the attribute with "ldap/" - for example, a value of "location" here would result in discovered desktops having a label with key "ldap/location" and the value being the value of the "location" attribute.|[]string|
|labels|Static labels applied to all hosts discovered via this policy.|map[string]string|
|rdp_port|The port to use for RDP for hosts discovered with this configuration. Optional, defaults to 3389 if unspecified.|number|
