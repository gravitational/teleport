|ssh_public_addr|Sets the hostport the proxy advertises for the SSH endpoint. The hosts in PublicAddr are included in the list of host principals on the SSH certificate.||
|trust_x_forwarded_for|Enables the service to take client source IPs from the "X-Forwarded-For" headers for web APIs received from layer 7 load balancers or reverse proxies.||

## ACME

Configures ACME protocol - automatic X.509 certificates

Example:

```yaml
enabled: "string"
email: "string"
uri: "string"
```
|Field Name|Description|Type|
|---|---|---|
|email|The email that will receive problems with certificate renewals|string|
|enabled|Whether ACME should be enabled|string|
|uri|ACME server URI|string|

## Kube Proxy

A `kubernetes` section in `proxy_service`.

Example:

```yaml
enabled: "string"
listen_addr: "string"
public_addr: # See description
kubeconfig_file: "string"
cluster_name: "string"
```
|Field Name|Description|Type|
|---|---|---|
|cluster_name|The name of a kubernetes cluster this proxy is running in. If set, this proxy will handle kubernetes requests for the cluster.|string|
|enabled||string|
|kubeconfig_file|An optional path to kubeconfig file, if specified, teleport will use API server address and trusted certificate authority information from it|string|
|listen_addr||string|
|public_addr|A publicly advertised address of the kubernetes proxy||

## UIConfig

Provides config options for the web UI served by the proxy service.

Example:

```yaml
scrollback_lines: 1
show_resources: # See description
```
|Field Name|Description|Type|
|---|---|---|
|scrollback_lines|The max number of lines the UI terminal can display in its history|number|
|show_resources|Determines which resources are shown in the web UI. Default if unset is "requestable" which means resources the user has access to and resources they can request will be shown in the resources UI. If set to `accessible_only`, only resources the user already has access to will be shown.||
