```yaml
  - "string"
  - "string"
  - "string"
```
|Field Name|Description|Type|
|---|---|---|
|auto_enroll|The toggle for the device auto-enroll feature.|string|
|ekcert_allowed_cas|An allow list of EKCert CAs. These may be specified as a PEM encoded certificate or as a path to a PEM encoded certificate.  If present, only TPM devices that present an EKCert that is signed by a CA specified here may be enrolled (existing enrollments are unchanged).  If not present, then the CA of TPM EKCerts will not be checked during enrollment, this allows any device to enroll.|[]string|
|mode|The trusted device verification mode. Mirrors types.DeviceTrust.Mode.|string|

## PKCS 11

Configures a PKCS#11 HSM to be used for private key generation and storage.

Example:

```yaml
module_path: "string"
token_label: "string"
slot_number: 1
pin: "string"
pin_path: "string"
max_sessions: 1
```
|Field Name|Description|Type|
|---|---|---|
|max_sessions|The upper limit of sessions allowed by the HSM.|number|
|module_path|The path to the PKCS#11 library.|string|
|pin|The raw pin for connecting to the HSM. Set this or PINPath to set the pin.|string|
|pin_path|A path to a file containing a pin for connecting to the HSM. Trailing newlines will be removed, other whitespace will be left. Set this or Pin to set the pin.|string|
|slot_number|The slot number of the HSM token to use. Set this or TokenLabel to select a token.|number|
|token_label|The CKA_LABEL of the HSM token to use. Set this or SlotNumber to select a token.|string|
