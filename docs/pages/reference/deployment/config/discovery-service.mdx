|Field Name|Description|Type|
|---|---|---|
|access_graph|Used to configure the cloud sync into AccessGraph.|[Access Graph Sync](#access-graph-sync)|
|aws|Used to match EC2 instances|[][AWS Matcher](#aws-matcher)|
|azure|Used to match Azure resources.|[][Azure Matcher](#azure-matcher)|
|discovery_group|The name of the discovery group that the current discovery service is a part of. It is used to filter out discovered resources that belong to another discovery services. When running in high availability mode and the agents have access to the same cloud resources, this field value must be the same for all discovery services. If different agents are used to discover different sets of cloud resources, this field must be different for each set of agents.|string|
|enabled||string|
|gcp|Used to match GCP resources.|[][GCP Matcher](#gcp-matcher)|
|kubernetes|Used to match services inside Kubernetes cluster for auto discovery|[][Kubernetes Matcher](#kubernetes-matcher)|
|listen_addr||string|
|poll_interval|The cadence at which the discovery server will run each of its discovery cycles. Default: [github.com/gravitational/teleport/lib/srv/discovery/common.DefaultDiscoveryPollInterval]||

## AWS Matcher

Matches AWS EC2 instances and AWS Databases

Example:

```yaml
types: 
  - "string"
  - "string"
  - "string"
regions: 
  - "string"
  - "string"
  - "string"
assume_role_arn: "string"
assume_role_name: "string"
external_id: "string"
tags: 
  "string": # See description
  "string": # See description
  "string": # See description
install: # [...]
ssm: # [...]
integration: "string"
kube_app_discovery: true
setup_access_for_arn: "string"
organization: # [...]
```
|Field Name|Description|Type|
|---|---|---|
|assume_role_arn|The AWS role to assume for database discovery.|string|
|assume_role_name|The AWS role name to assume for discovery.|string|
|external_id|The AWS external ID to use when assuming a role for database discovery in an external AWS account.|string|
|install|Sets the join method when installing on discovered EC2 nodes|[Install Params](#install-params)|
|integration|The integration name used to generate credentials to interact with AWS APIs. Environment credentials will not be used when this value is set.|string|
|kube_app_discovery|Controls whether Kubernetes App Discovery will be enabled for agents running on discovered clusters, currently only affects AWS EKS discovery in integration mode.|Boolean|
|organization|An AWS Organization matcher for discovering resources across multiple accounts under an Organization.|[AWS Organization Matcher](#aws-organization-matcher)|
|regions|AWS regions to query for databases.|[]string|
|setup_access_for_arn|The role that the discovery service should create EKS Access Entries for.|string|
|ssm|Provides options to use when sending a document command to an EC2 node|[AWS SSM](#aws-ssm)|
|tags|AWS tags to match.|map[string]|
|types|AWS database types to match, "ec2", "rds", "redshift", "elasticache", or "memorydb".|[]string|

## AWS Organization Units Matcher

Contains rules for matching accounts under an Organization. Accounts that belong to an excluded Organizational Unit, and its children, will be excluded even if they were included.

Example:

```yaml
include: 
  - "string"
  - "string"
  - "string"
exclude: 
  - "string"
  - "string"
  - "string"
```
|Field Name|Description|Type|
|---|---|---|
|exclude|A list of AWS Organizational Unit IDs and children OUs to exclude. Accounts that belong to these OUs, and their children, will be excluded, even if they were included. Only exact matches are supported. Optional. If empty, no OUs are excluded.|[]string|
|include|A list of AWS Organizational Unit IDs and children OUs to include. Accounts that belong to these OUs, and their children, will be included. Only exact matches or wildcard (*) are supported. Required.|[]string|

|Field Name|Description|Type|

|sshd_config|Provides the path to write sshd configuration changes|string|
|suffix|Indicates the installation suffix for the teleport installation. Set this value if you want multiple installations of Teleport. See --install-suffix flag in teleport-update program.|string|
