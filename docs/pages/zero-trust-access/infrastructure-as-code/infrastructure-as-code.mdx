>
  There are two ways to configure a Teleport cluster:
 
  - **Static configuration** passes options to a single Teleport process, such as an agent or Teleport Proxy Service instance.
  - **Dynamic resources** define role-based access controls and protected infrastructure, such as users, roles, and clusters. They are stored by the Auth Service and can be updated using the `tctl` CLI or infrastructure-as-code tools.
</DocHero>
