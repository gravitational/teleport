Teleport logs cluster activity by emitting various events into its audit log.

This can be viewed in your Teleport Cluster by clicking on the **Audit** tab in the left sidebar and choosing **Audit Log**.

Here you'll see events like successful user logins along with metadata like remote IP address, time, and the session ID.

![View audit log](../../img/get-started/get-started-audit-log@2x.png)

In addition to viewing the audit log, every SSH, desktop, or Kubernetes shell session is recorded and can be replayed. 

These can be viewed by clicking on the **Audit** tab in the left sidebar and choosing **Session Recordings**. There you can click **Play** to see a full recording of a session.

For a more in-depth look at how Teleport's audit system works, including event types and descriptions, storage options, and exporting events, see our [Audit Events and Records guide](../reference/monitoring/audit.mdx).
