Teleport logs all cluster activity as structured audit events and captures full session recordings for SSH, desktop, and Kubernetes shell sessions. This gives you a tamper-evident record of what users did, when, and from where.

## View audit events in the Web UI

To see audit events, click **Audit** > **Audit Log** in the left sidebar. You will see events like successful logins, session starts, and configuration changes, each with metadata including event type, remote IP address, timestamp, and the identity involved. Click **Details** to see the full event in JSON format.

![View audit log](../../img/get-started/get-started-audit-log@2x.png)

## View session recordings

Teleport records SSH, desktop, and Kubernetes shell sessions. To replay a session:

1. Click **Audit** > **Session Recordings** in the left sidebar.
2. Find the session you want to review.
3. Click **Play** to watch a full playback of the session.

This is useful for security audits, compliance reviews, and post-incident investigation.

## [Advanced] Export audit events

For long-term retention, compliance reporting, or integration with your SIEM, you can export Teleport audit events to an external log management platform using the **Event Handler** plugin.

The Event Handler reads events from Teleport and forwards them via Fluentd to your destination of choice. If you are new to audit event export, start with [Forwarding Events with Fluentd](../zero-trust-access/export-audit-events/fluentd.mdx), which explains how the plugin works and how to run it.

From there, read the guide for your log platform:

- [Elastic Stack](../zero-trust-access/export-audit-events/elastic-stack.mdx)
- [Splunk](../zero-trust-access/export-audit-events/splunk.mdx)
- [Datadog](../zero-trust-access/export-audit-events/datadog.mdx)
- [Panther](../zero-trust-access/export-audit-events/panther.mdx)

## [Advanced] Enable enhanced session recording

Standard session recording captures terminal output. Enhanced session recording goes further: it uses Linux BPF to capture the actual commands executed by the user, including commands run inside shell scripts or through obfuscation techniques.

This provides a stronger audit trail for compliance use cases where you need a record of commands, not just output.

Read [Enhanced Session Recording for SSH with BPF](../enroll-resources/server-access/guides/bpf-session-recording.mdx) to enable it on your enrolled Linux servers.
