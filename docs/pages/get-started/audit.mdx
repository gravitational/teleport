Teleport logs cluster activity by emitting various events into its audit log.

This can be viewed in the Teleport Web UI by clicking on **Audit** > **Audit Log** in the left sidebar.

Here you'll see events like successful logins along with metadata such as event type, remote IP address, timestamp, and the identity involved. Click on **Details** to see the complete event information in JSON format.

![View audit log](../../img/get-started/get-started-audit-log@2x.png)

## View your SSH session recording

In addition to recording structured events in its audit log, Teleport can also capture full session recordings for SSH, desktop, or Kubernetes shell sessions.

Remember when you connected to your Ubuntu server via SSH in Step 2? That entire session was recorded.

To view the recording:

1. Click on **Audit** > **Session Recordings** in the left sidebar
2. Find your recent SSH session to your Ubuntu server
3. Click the **Play** button to watch a full playback of everything you typed and the server's responses

This is especially useful for security audits, compliance requirements, and troubleshooting.

For a more in-depth look at how Teleport's audit system works, including event types and descriptions, storage options, and exporting events, see our [Audit Events and Records guide](../reference/deployment/monitoring/audit.mdx).

## Optional: Set up advanced features

With additional configuration, you can set up Teleport to get even more detailed
insights into activity within your cluster, helping you detect potential
vulnerabilities or investigate a security incident. 

After you finish getting started with Teleport, you can plan to complete these
advanced setup steps:

1. **Enhanced session recording** uses BPF to capture activity on
   Teleport-protected servers directly from the kernel. You can enable enhanced
   session recording using the Teleport configuration file of a Teleport SSH
   Service instance like the one you enrolled in [Step 2](connect.mdx). First,
   you'll need to make sure that the server you are enrolling meets certain
   criteria. To get started, read [Enhanced Session Recording for SSH with
   BPF](../enroll-resources/server-access/guides/bpf-session-recording.mdx).

1. You can set up the **Teleport Event Handler** to export audit events from the
   Teleport Auth Service to a third-party platform like Splunk or Datadog.  To
   get started, you will need to deploy the Teleport Event Handler, either by
   installing a binary on a Linux server or installing a Helm chart on a
   Kubernetes cluster. Read [Forwarding Events with
   Fluentd](../zero-trust-access/export-audit-events/fluentd.mdx) to learn the
   basics of how the Event Handler plugin works.
